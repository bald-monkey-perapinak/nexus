## ✦ LangGraph — детальный разбор графов

### Graph 1 · Onboarding

```
START ──► validate_input ──► enrich_profile ──► calculate_completeness ──► END
                │ (missing fields)
                └──────────────────────────────────────────────────────► END
```

| Узел | Тип | Описание |
|------|-----|----------|
| `validate_input` | детерм. | Проверяет 4 обязательных поля |
| `enrich_profile` | LLM | Llama-70b нормализует опыт, выводит `city_tier`, `implied_segment` |
| `calculate_completeness` | детерм. | Взвешенный скор 0–100, влияет на качество генерации |

### Graph 2 · Idea Generation

```
                         ┌─► financial_discriminator ─┐
generate_candidates ───► ├─► market_discriminator     ─┼─► aggregate_scores ──► enrich_cards ──► END
     (llama-70b)         └─► ops_discriminator        ─┘   (deterministic)      (llama-8b)
                              (all 3 in parallel,
                               llama-8b-instant)
```

**Правила агрегации скоров:**

```
Любой verdict = "fail"   →  ❌ идея отсеяна
Два verdict  = "warn"    →  ❌ идея отсеяна
Один verdict = "warn"    →  ⚠️  идея проходит с меткой
Итоговый скор = fin×0.4 + mkt×0.4 + ops×0.2
Топ-6 по итоговому скору → финальные карточки
```

### Graph 3 · Financial Model

```
generate_base_assumptions ──► calculate_scenarios ──► generate_unit_economics ──► validate_model ──► END
       (llama-70b)               (pure math)               (llama-70b)             (deterministic)
```

**Три сценария из одной базы:**

| Сценарий | Чек × | Клиенты × |
|----------|-------|-----------|
| Пессимистичный | 0.7 | 0.5 |
| Базовый | 1.0 | 1.0 |
| Оптимистичный | 1.3 | 1.75 |

## ✦ Быстрый старт

### Требования

- Docker & Docker Compose
- [Groq API key](https://console.groq.com/keys) (бесплатно)
- Telegram Bot Token от [@BotFather](https://t.me/BotFather)
- [ngrok](https://ngrok.com) для локальной разработки

### 1 · Клонирование

```bash
git clone https://github.com/bald-monkey-perapinak/nexus.git
cd nexus
```

### 2 · Переменные окружения

```bash
cp .env.example .env
```

```dotenv
# .env — минимальный набор для запуска
BOT_TOKEN=7xxxxxxxxx:AAxxxxxxxxxxxxxxxxxxxxxxxx
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
WEBAPP_URL=https://xxxx-xx-xx-xxx-xx.ngrok-free.app
JWT_SECRET=замените-на-случайную-строку-32-символа
```

### 3 · Запуск через Docker

```bash
# Поднять всё одной командой
docker compose up -d

# Проверить статус
docker compose ps

# Логи бэкенда
docker compose logs -f backend
```

Сервисы после запуска:

| Сервис | URL |
|--------|-----|
| Frontend | http://localhost:4000 |
| Backend API | http://localhost:8001 |
| API Docs | http://localhost:8001/docs |

### 4 · Локальная разработка (без Docker)

```bash
# Backend
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# Frontend (новый терминал)
cd frontend
npm install
npm run dev           # → http://localhost:5173 (dev server Vite)

# Bot (новый терминал)
cd bot
pip install python-telegram-bot python-dotenv
python bot.py
```

### 5 · Подключение к Telegram

```bash
# Пробросить dev-сервер фронта наружу (для локальной разработки с ngrok)
ngrok http 5173

# Или, если используете Docker — пробросить контейнер фронта
ngrok http 4000

# Скопировать HTTPS-URL в .env → WEBAPP_URL
# Перезапустить бот
```

```
/start → бот покажет кнопку [🔷 Открыть Nexus] → MiniApp открывается внутри Telegram
```

```
ИТОГО на полный цикл                ~8–12s
```
